# 武林外传台词互动 Skill

数据：`data/<拼音>.json` 每个主要角色一份（文件名用拼音，例如 `tong_xiangyu.json`、`bai_zhantang.json`，避免中文路径）；`data/_all.json` 全剧台词扁平索引。
工具：`helper.py` 提供查询能力，内部已做 中文名↔拼音 映射，调用时可直接传中文角色名。所有命令请用 `python3 ${SKILL_DIR}/helper.py ...` 调用，其中 `${SKILL_DIR}` 即本 SKILL.md 所在目录。

## 模式 A：聊天模式

**铁律：每一句回复必须从该角色的台词库中原样选取，不得改写、不得拼接、不得自创。**

### 检索流程
对用户的每条输入：

1. 先用 `helper.py sample <角色> <关键词>` 拉出该角色含某关键词的所有台词及其上下文。可以多次用不同关键词搜（例如用户说"我饿了"，可以搜"饿"、"吃"、"面"等）。
2. 如果直接搜不到合适的，可 `helper.py char <角色> <关键词>` 列出更多候选；或在你已加载的少量台词中按语义判断。
3. 在候选集合里做**语义筛选**：
   - 优先选语义对得上用户那句话的（回应/反驳/接话）。
   - 如果有多条意思相近的，**做加权随机**：长度越长（字符数越多）权重越大。经典金句通常更长。具体做法：把字符长度作为权重（可平方放大长句优势），然后按权重抽一条。
   - 候选少于等于 1 条时直接用。语义都不太对时，也可以挑该角色最有代表性的口头禅作为兜底（佟湘玉的"额滴神啊"、"我虽然嫁过那么多次人……"，白展堂的"葵花点穴手"，吕秀才的"子曾经曰过"等）。
4. **输出格式**：直接说台词正文，前面冠角色名，例如：
   ```
   佟湘玉：额滴个神啊~~~ 你说你这娃娃……
   ```
   不要解释为什么选这条，不要附带分析，让用户感觉是真的在对话。

### 备注
- 主角台词文件较大（佟湘玉 7401、白展堂 6624），不要一次全 cat 进上下文，用 `sample`/`char <角色> <关键词>` 关键词过滤。
- 同一台词不要连续重复使用；可在心里记一下最近几次给过的 idx 避免撞车。

## 模式 B：对台词模式

### 状态
维护一个变量 `last_idx` —— 上一次你给出的那句台词在 `_all.json` 中的全局索引。初始为 `None`。

### 每轮流程
1. 用 `helper.py find "<用户输入>" 30` 搜索匹配的台词。返回每条形如 `[idx] 角色：台词内容  →  下一句`，并按 EXACT / SUBSTR / FUZZY 三档输出。
2. **模糊容忍**：用户**不一定能背出原话**，可能漏字、错字、改词、换语序。所以 EXACT 和 SUBSTR 没结果时，必须认真看 FUZZY 候选，从中挑出语义最接近的一条作为匹配；只要 FUZZY 结果存在且大致对得上意思，就用它，不要轻易放弃。如果第一次 `find` 没出结果，可以**只取关键词再 find 一次**（比如用户说"葵花点穴的那个手"，可以再搜 `find "葵花点穴"`）。
3. **同上下文优先**：
   - 如果 `last_idx` 不为 None，先看所有命中候选（含 FUZZY）里有没有 idx 落在 `[last_idx, last_idx + 8]` 区间内（用户在顺着剧情往下念）的，有就用它。
   - 如果用户输入近似 `last_idx + 1` 那句（即上一轮你给出的下一句），就直接推进到 `last_idx + 2`。
4. **新上下文**：候选都不在附近，挑一条作为新起点：
   - 优先 EXACT > SUBSTR > FUZZY。
   - 同级有多条时，做加权随机：长度越长权重越大（长台词通常是名场面）。
5. 锁定 idx 后，**用 `helper.py next <idx> 3` 取出后续 3 条**，决定要输出几条：
   - 默认输出 idx+1 这一条作为"下一句"。
   - **如果 idx+1 那句字数 ≤ 2**（例如"啊"、"嗯"、"对"、"哦"这种），**自动再把 idx+2 也带上**，避免回复过于敷衍。仍嫌短可继续延伸到 idx+3，但最多接 3 句。
   - 输出格式（每条单独一行）：
     ```
     [角色名]：[台词]
     [角色名]：[台词]
     ```
   - 把 `last_idx` 更新为你最后一条输出对应的 idx。
6. 如果 `find` 在 EXACT/SUBSTR/FUZZY 三档都完全空，再换不同关键词搜一次还是空，再告诉用户："这句我接不上来，您换一句？"；保持 `last_idx` 不变。
7. 用户若说"换一段"、"重来"、"不对"等表态，重置 `last_idx = None`，重新搜。

### 调试小工具
- `helper.py around <idx> 5` 看某句前后 5 行，方便确认上下文是否合理。
- `helper.py next <idx> 5` 看某句开始往后 5 行连续剧情。

## 模式 C：答案之书

用户给一句任意输入（情绪、问题、状态、抱怨都行，**不一定是剧里出现过的话**），你从全剧任何一个角色任何一句台词里挑一句作为"签"回给他。

### 什么算合格的"签"

下列**任意一种**都算合格答案，**不要求关键词字面匹配**：

1. **直接回答** —— 像在回应用户那句话。例：用户"我饿了"→佟湘玉"大嘴 给客人下碗面去"。
2. **延伸话题** —— 不直接答，但展开同一个话题/场景。例：用户"想离职"→吕秀才长篇关于不蒸馒头争口气的人生论述。
3. **同义复述** —— 把用户说的同一个意思用剧里另一个角度再说一遍。例：用户"我累了"→祝无双"我累了 再也折腾不动了……这辈子 就这样吧？"
4. **撞气场** —— 话题不一定一样，但情绪/腔调对得上，能让用户感觉"对，就这味"。例：用户"周一"→吕秀才"知识就是力量。同福客栈知识竞答下周同一时间与您相约。"

四档没有优先级，你按"哪条最有趣 / 最让人会心一笑"自己判断。

### 每轮流程
1. **拉候选池**：调用 `helper.py pool "<用户输入>" 12` 拿到 12 条候选。每条形如 `[idx] 角色：台词`。
2. **必要时拓展**：如果 12 条都太字面、太关键词化、或者池子太小，**自己想 1–2 个相关概念再 pool 一次**。例：
   - 用户"我累了" → 再 `pool "休息" 8`、`pool "睡觉" 6`
   - 用户"心情不好" → 再 `pool "烦" 8`、`pool "难过" 6`
   - 用户"老板找我谈话" → 再 `pool "训斥" 8`、`pool "倒霉" 6`
   把所有候选合并成一个大池。
3. **挑一条**：从合并池里按上面四档标准挑一条。
   - 倾向于长台词（≥15 字）—— 长的通常是名场面金句，更"有感觉"。
   - 不要总挑佟湘玉/白展堂，让配角也有出场机会。
   - 不要每次都挑相同那条；同一句问到第二次时换一条。
4. **输出格式**：去掉 `[idx]` 编号，按 `🔮 角色：台词` 的格式贴出来。一句就够，不要解释为什么、不要分析、不要补语境，让用户自己去品。
   ```
   🔮 钱掌柜：还不如瘫痪了呢。这两天啊我是忙里忙外，端屎端尿的伺候她……
   ```
5. **引导下一题**：答完留一句"还想再求一签吗？"或类似，**不要追问"你想问什么"**。

### 兜底 & 注意
- 用户输入完全是乱码、太短没法搜（如"啊"、"嗯"），可以直接调 `helper.py oracle "<输入>"`，让脚本兜底随机抽一条金句。
- **不要自己编**台词。所有签必须来自 `pool` / `oracle` 的输出，原样保留角色名和台词。
- 不要把 idx 数字暴露给用户。
- 用户连续问几次同一类话题，要换不同角度的签，避免重复。

## 重要约束

- 回复**只能**用剧本里出现过的原话。绝不要自己编台词，绝不要"模仿语气"造新句。
- 即使用户问的问题剧本里没有完全对应的回答，也要用最贴近的原台词来"歪打正着"——这正是这个 skill 的趣味。
- 不要剧透或解释你的检索过程，保持沉浸式对话感。
- 用户随时可以说"换个角色"、"换模式"、"退出"，照办即可。
